## 🗺️ Cartographie de l'Espace de Travail (Map)

Ce dépôt centralise l'ensemble des travaux pratiques réalisés dans le cadre du module de **Génie Logiciel**. L'architecture a été pensée de manière modulaire, chaque dossier représentant un domaine de compétence technique et organisationnel spécifique.

```text
📦 Mes_Tps (Racine du Dépôt)
 ┣ 📂 TP_docker      🐳 Orchestration multi-conteneurs (Nginx, PostgreSQL, pgAdmin)
 ┣ 📂 Poo            ☕ Programmation Orientée Objet (Encapsulation, Héritage, Polymorphisme)
 ┣ 📂 Tp_Git         🔀 Maîtrise avancée du versionnement (Merge, Rebase, Cherry-pick)
 ┣ 📂 Tp_Notion      📋 Gestion de projet Agile (Kanban synchronisé avec le workflow Git)
 ┗ 📜 README.md      📖 Ce fichier de présentation globale
```

---

## 🎯 Vue d'Ensemble des Modules

### <span style="color:#2496ED">1. Infrastructure & Déploiement (Docker)</span>
Mise en place d'une architecture de microservices isolée et persistante via **Docker Compose**.
- Configuration d'un serveur Web frontend.
- Intégration d'une base de données relationnelle sécurisée.
- Attachement de volumes pour la persistance des données.

### <span style="color:#ED8B00">2. Algorithmique & Architecture (POO Java)</span>
Application stricte des paradigmes de la Programmation Orientée Objet.
- Conception de classes abstraites et gestion rigoureuse de l'état interne.
- Déploiement du polymorphisme sur divers systèmes (Simulation zoologique, Flotte de véhicules, Système bancaire).

### <span style="color:#F05032">3. Contrôle de Version Avancé (Git & Node.js)</span>
Simulation d'un environnement de collaboration intensif sur un projet JavaScript.
- Création et gestion de branches isolées (`feature`, `bugfix`).
- **Résolution manuelle de conflits** de fusion (Merge Conflicts).
- Optimisation et réécriture de l'historique via `git rebase` et `git cherry-pick`.

### <span style="color:#000000">4. Gestion de Projet Collaborative (Notion)</span>
Alignement entre la méthodologie Agile et le développement logiciel.
- Conception d'un espace de travail **Notion** avec vue Kanban.
- Synchronisation parfaite entre les mouvements de tickets Kanban et les fusions de branches Git.
- Traçabilité totale (Documentation des conflits, Réunions de sprint, Liens vers les commits).
